2. Usability Test

The usability evaluations focuses on measuring efficiency, effectiveness, and user satisfaction.

  • Effectiveness focuses on errors and task completion rates, categorized as “complete,” “complete with errors,” or “incomplete.”
  • Efficiency focuses on comparing the time needed to complete the safeguarding and verifying workflows.
  • Usability testing is based on Jakob Nielsen’s 10 usability heuristics — a set of high-level guidelines based on an
    understanding of human behavior, psychology, and information processing.

Effectiveness Test

The plugin allows these safeguarding functions to be carried out in combination or independently. The combined mode enforces an ordered execution of safeguarding functions, while the individual mode gives users the flexibility to perform any function first.

Combined mode - should be already chosen by default

  • Load the dataset - it should be in the dataset folder of the plugin zip file.
  • Carry our geomasking and save the layers.
  • Close the plugin, and select the original layer again. Otherwise it selects the masked layer and hexabins that instead of the original layer (working on resolving this).
  • Select the original layer.
  • Carry out hexabining, and save layers
  • Encrypt one, two, or three levels. If choosing three layers, the second and third layers would be from the saved masked or hexabinned layers.
  • Notarise, and open the transaction link in Sepolia Testnet Explorer using the link displayed in the plugin.
Individual Mode
  • You can start from any security function and thereafter choose any function (moving to the right) as the next option. The original dataset can be first masked, encrypted, or notarised, without having to go in that order.
  • Thereafter, the next security function (masking or encryption) can be skipped as well. For instance, users can mask the dataset, but avoid encryption, and move on to notarise the masked dataset.

When undertaking these safeguarding an verification tasks, fill out the following form. This should not take more than an hour.

When using the MapSafe plugin, you’ll find its key feature is the simplicity of its interface and the thorough documentation provided. The interface is clean and intuitive, with explanatory tooltips available next to nearly every button for functions like masking, hexabinning, encryption, notarization, verification, and decryption. These tooltips help guide you through each step.

As you work, you’ll see the original, masked, and hexabinned datasets displayed as QGIS layers. This allows you to visually compare the datasets before and after anonymization.

For additional guidance, refer to the this plugin’s help website, which provides a basic guide covering all features. The guide explains how the plugin works, the trade-offs involved (e.g., deciding on masking distances), and its limitations. The above detailed video walkthrough demonstrates the safeguarding and verification processes from start to finish.

After downloading your anonymized dataset or encrypted volumes, the plugin will display reminders, such as not to reveal specific masking parameters to the public and to securely store the encryption passphrase. Finally, once you complete the notarization process, the plugin will display the URL where the encrypted dataset’s details are stored, ensuring you can access this information easily.
