Usability Test

1. Usability Test

The usability evaluation for the MapSafe QGIS plugin is based on Jakob Nielsen’s 10 usability heuristics — a set of high-level guidelines based on an understanding of human behavior, psychology, and information processing.

The Heuristic Evaluation workbook (in PDF) can be downloaded from this website: https://www.nngroup.com/articles/how-to-conduct-a-heuristic-evaluation/

2. Test Steps

Carrying out the test should not take more than an hour. 1. The first step would be to familiarise yourself with the plugins safeguarding and verification features via the following video. Also understand the Passphrase shielding feature.

This plugin's features are are demonstrated in a 5 minute video.

2. The plugin can be downloaded directly from QGIS, through the plugins menu: Plugins —> Manage and Install Plugins. This website serves as the help menu for the plugin.

Since geospatial data can be sensitive, the security processes within the plugin can be complex, it would be a bonus to carry out a usability evaluation before others start to use it. Thus, I would be grateful if you could test it as a user and give any feedback. A usability evaluation of the plugin is crucial before its rolled out for users and for our paper.

Complete the following steps to use the plugin.

  • Installing Python and the required libraries
  • MapSafe Plugin Settings
    • Individual or Combined Mode

    • Seting up for Notarisation
    • Help Menu

1. Installing Python and the required libraries

Python can be installed on the machine. The The Windows installer can be found here

The required Python libraries can be installed via pip. To use pip in a Windows environment, one needs to run the OSGeo4W.bat from the 'D:\QGIS' directory (not from 'D\OSGEO4W').

These commands need to be issued using the D:\QGIS\OSGEO.bat
D:\QGIS>pip install cryptography
D:\QGIS>pip install web3
D:\QGIS>pip install python-dotenv
D:\QGIS>pip install xkcdpass
D:\QGIS>pip install easygui or python -m pip install easygui
D:\QGIS>pip install python-dotenv
D:\QGIS>(python -m) pip install pyqt-switch

2. MapSafe Plugin Settings

From the plugin's main window, the following need to be set. Note only the working directory is necessary, if one does not need to perform notarisation.



2 (a). Individual or Combined Mode

The plugin can be used in two modes. The combined mode (default) enforces and guides the user in perfoming these three functions in order in a workflow. The individual mode allows users to use the plugin's three security functions (masking or hexabinning, encryption and notarisation) individually or in any combination.

This selection can be made in the switch at the top left (see above image). The figure below outlines the different combinations in which the security functions in the individual mode can be used. These can be used collectively to fully safeguard their data, or independently, for example, users can choose to only mask their maps. They can skip masking and only encrypt it and download the encrypted volume. After encryption, they have an option to further notarise it.



2 (b) Seting up the Working Directory

It is essential for users to set the working directory, where the resulting masked, binned, and encrypted datasets are Saved. This directory is also used by the plugin to perform computations and hold temporary data.

Users need to set this directory in the env variables dialog the first time the plugin is loaded (see above image).

2 (c) Seting up for Notarisation

Setting these details is not necessary if one does not need to perform notarisation.

Ideally, all information should be notarised on the Ethereum mainnet. However, one can use the public Ethereum testnet, Sepolia for testing. These are the steps needed for minting in Sepolia testnet:

  • First, visit Vanity-ETH to generate an Ethereum vanity address and a private key.
  • Second, using your generated Ethereum address, request Ether from this Goerli Faucet
  • If using the testnet, you can check the Ether balance and all minted data on your address, at the Goerli etherscan
  • Finally, set the private key and testnet information using the Set Variables button on the plugin's top right (see above image).

2 (d) Help

The Help button directs the users to this website.

3. Usability Test procedure

The test procedure is the same steps carried out in the YouTube video. https://www.youtube.com/watch?v=gq4wX3kMTfE You can also try encrypting any number of levels.

Choose combined mode - should be already chosen by default.

  • Load the dataset - it should be in the dataset folder of the plugin zip file.
  • Carry our geomasking and save the layers.
  • Close the plugin, and select the original layer again. Otherwise it selects the masked layer and hexabins that instead of the original layer (working on resolving this).
  • Select the original layer.
  • Carry out hexabining, and save layers
  • Encrypt one, two, or three levels. If choosing three layers, the second and third layers would be from the saved masked or hexabinned layers.
  • Notarise, and open the transaction link in Sepolia Testnet Explorer using the link displayed in the plugin.
Individual Mode

  • You can start from any security function and thereafter choose any function (moving to the right) as the next option. The original dataset can be first masked, encrypted, or notarised, without having to go in that order. Thereafter, the next security function (masking or encryption) can be skipped as well. For instance, users can mask the dataset, but avoid encryption, and move on to notarise the masked dataset. This is a useful option in our plugin. However, it can be complicated to understand, and therefore, we have created two modes, with the combined mode as default.
